CXXRTL DUT Implementation Requirements
============================================================

You MUST generate a file equivalent to `dut.cc` with the following rules:

1. You MUST use the `cxxrtl` namespace types and define a struct named

2. The struct MUST be STANDALONE. Do NOT inherit from `cxxrtl::module`

3. Port naming convention (CXXRTL name mangling):
   - Each Verilog port `foo` becomes a member named `p_foo`
   - Underscores in Verilog names are doubled: `my_sig` -> `p_my__sig`

4. Port types:
   - Input ports:  `cxxrtl::value<WIDTH> p_name;`
   - Output ports: `cxxrtl::value<WIDTH> p_name;`
   - For sequential circuits with clk:
       - `cxxrtl::wire<1> p_clk;`

5. Required methods (plain, non-virtual, NO `override`):
   - `void eval()`  — compute outputs from inputs (and update state)
   - `bool commit()` — may simply `return false;`

6. Combinational vs Sequential logic:
   - If the circuit is combinational:
       - Do NOT store any state
       - `eval()` computes outputs purely from current inputs
       - `commit()` returns false
   - If the circuit is sequential:
       - Maintain state using member variables (plain `cxxrtl::value<N>`)
       - `p_clk` is a `cxxrtl::wire<1>`
       - The test harness drives BOTH `p_clk.curr` and `p_clk.next` to the
         current clock level, then calls `eval()` followed by `commit()`
         EXACTLY ONCE per clock level (once with clk=1, once with clk=0
         per cycle)
       - Therefore, treat `p_clk.curr.data[0] & 1` being true inside
         `eval()` as the rising clock edge: update all registers there
       - When several registers update on the same edge, compute every
         next-state value from the CURRENT state before assigning any of
         them (registers update simultaneously in hardware)
       - After updating (or not updating) state, always drive the output
         ports from the state
       - Do NOT call `p_clk.commit()` — `commit()` may simply return false

7. Bit-width behavior:
   - Use `cxxrtl::value<N>` for N-bit values
   - Access raw data via `.data[i]` (array of uint32_t chunks)
   - For values <= 32 bits, use `.data[0]`
   - Apply bit-masks when needed

8. You MUST include the CXXRTL header:

Example (Sequential — 8-bit D flip-flop with reset)
============================================================

```cpp
#include <cxxrtl/cxxrtl.h>

namespace cxxrtl_design {

struct p_TopModule {
    cxxrtl::wire<1> p_clk;
    cxxrtl::value<1> p_reset;
    cxxrtl::value<8> p_d;
    cxxrtl::value<8> p_q;

    cxxrtl::value<8> state_q;

    void eval() {
        // The harness calls eval() exactly once per clock level, so a high
        // clock level here corresponds to the rising edge of clk.
        bool posedge_clk = p_clk.curr.data[0] & 1;

        if (posedge_clk) {
            if (p_reset.data[0] & 1) {
                state_q.data[0] = 0;
            } else {
                state_q.data[0] = p_d.data[0] & 0xFF;
            }
        }

        p_q.data[0] = state_q.data[0] & 0xFF;
    }

    bool commit() {
        return false;
    }
};

} // namespace cxxrtl_design
```
